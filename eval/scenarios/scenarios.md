# Eval scenarios (v1)

The first scenario set for the simulated-scientist eval loop. Eight scenarios
across four levels, sharing **one coherent dataset universe** so they reuse
data and build on each other, and so the data carries **planted ground truth**
the eval can check.

## Design principles

1. **One study, one data family.** Everything lives in a monocyte/macrophage
   stimulation study (control vs stimulated, several donors, a few timepoints,
   an interferon-stimulated-gene response). This matches the platform's existing
   `cells.csv` schema and the mockup's "Macrophage Stimulation Pilot."
2. **Planted ground truth.** Each dataset encodes specific, known facts (a
   bad-QC donor, a robust ISG effect, a fragile single gene, a *trap* gene that
   is a pure artifact). Scenarios are "solved" when the agent discovers/handles
   these correctly — that's what probes and rubrics check against.
3. **Small data.** All CSVs are synthetic, deterministic (fixed RNG seed), and
   total **< 150 KB**. No real downloads required for v1.
4. **Live vs pre-seeded.** L1–L3 run *live* (the simulated scientist actually
   drives analyses). L4 scenarios operate on a **pre-seeded project graph**
   (entities + provenance fabricated by a seed script) plus one small live
   addition — so we get scale-at-rest without paying to generate 30 real plots.

## Datasets

Generated by `eval/scenarios/data/gen.py` (numpy, seed=7). Schemas below.

### `monocyte_stim.csv` — primary (workhorse for L1, L2; base for L3/L4)
Per-cell table. **6 donors (D1–D6) × 2 conditions (control, stim) × 3 timepoints
(0h, 2h, 6h) × ~15 cells ≈ 540 rows**, ~16 columns (~55 KB).

Columns: `cell_id, donor, condition, timepoint, n_genes, n_counts, mt_fraction,
IFIT1, IFIT3, ISG15, MX1, OAS1, CXCL10, ACTB, GAPDH, GENE_X`

**Planted ground truth:**
- **QC**: donor **D6** is low-quality — `mt_fraction` ≈ 0.13 (vs ≈ 0.04 others),
  lower `n_genes`. *(L1-QC, L1-DONOR)*
- **ISG response**: the ISG set (IFIT1/IFIT3/ISG15/MX1/OAS1) is elevated in
  **stim vs control**, rising with timepoint, **peak at 6h**. Strong in
  D1/D2/D4/D5, **weak in D3** (leave-one-donor-out–sensitive), noisy in D6.
  Module-level effect holds in **5/6 donors**. *(L2-IFN, L2-TIME)*
- **CXCL10**: strongly up in stim, but its significance **collapses when D3 is
  removed** — the single-gene caveat. *(L2-IFN)*
- **GENE_X (the trap)**: looks "up in stim" in a naïve test, but the elevation
  is driven **entirely by the contaminated donor D6**. Drop D6 (or control for
  QC) → the effect vanishes. *(L3-OVERTURN)*
- **ACTB/GAPDH**: flat across conditions (negative controls).

### `monocyte_ref.csv` — replication cohort (L3-REPL)
Independent cohort, donors **R1–R4**, control/stim, single timepoint (6h).
**~240 rows** (~25 KB). Same columns minus `timepoint` variation.
**Ground truth:** the **ISG module effect replicates robustly across all 4
donors**; `GENE_X` is **flat** here (confirming the L3-OVERTURN artifact).

### Stub datasets — `tcell_stim.csv`, `bcell_baseline.csv` (L4-SCALE only)
Tiny (~80 rows each), just enough to exist as distinct dataset entities so the
project spans **4 datasets**. Same QC columns + a couple of markers.

### Seed scripts (L4)
- `eval/scenarios/seed/submitted_manuscript.py` — fabricates a "submitted paper"
  project: datasets `monocyte_stim` + `monocyte_ref`, ~5 figures, ~3 results,
  3 findings, 2 claims, 1 manuscript narrative, full provenance edges. Used by
  **L4-REVIEW**.
- `eval/scenarios/seed/large_project.py` — fabricates a sprawling project:
  **4 datasets, ~30 figures, ~12 results, 6 findings, 3 claims, a manuscript**,
  with provenance. Figures are entity stubs (title + edges, tiny/no artifact) —
  enough for navigation/clutter/provenance probes. **One claim deliberately has
  a missing evidence link** (target for the audit probe). Used by **L4-SCALE**.

## Scenarios

Each: **goal** (the scientist's intent the driver pursues), **seed** (starting
state), **milestones** (expected artifact trajectory), **exercises** (which
metrics / probes / perception tiers), **ground-truth checks** (pass criteria),
**cost**.

### L1-QC — QC triage
- **Goal:** "I just got this scRNA-seq table — is the data usable, and is any
  sample bad?"
- **Seed:** empty project; upload `monocyte_stim.csv`.
- **Milestones:** focus dataset → plot `mt_fraction` & `n_genes` distributions →
  notice D6 is off → highlight/keep the QC figure (pin).
- **Exercises:** basic flow; pin gesture; Tier-1 audits on the dataset/figure
  views; actions-to-first-kept-figure.
- **Ground truth:** agent identifies **D6** as low-quality (high mt_fraction);
  ≥1 QC figure pinned.
- **Cost:** light (≈3–5 runs).

### L1-DONOR — donor effect check
- **Goal:** "Does mitochondrial fraction differ by donor? I want a clean result
  I can keep."
- **Seed:** `monocyte_stim.csv` loaded.
- **Milestones:** groupby donor → barplot of mean `mt_fraction` → one-line read →
  **promote figure → result**.
- **Exercises:** promote-to-result flow; suggest-interpretation pre-fill quality;
  effort to create a result.
- **Ground truth:** result correctly states **D6 is the outlier**; a `result`
  entity exists with the figure as evidence.
- **Cost:** light (≈3–4 runs).

### L2-IFN — monocyte interferon response
- **Goal:** "Characterize the stimulation response in monocytes and decide if
  it's a real finding."
- **Seed:** `monocyte_stim.csv` loaded (ideally after L1 in a shared project).
- **Milestones:** subset/aggregate → pseudobulk-style DE stim vs control →
  ISG-module score figure → leave-one-donor-out robustness → **draft a finding**
  ("stimulation induces an ISG-high state") with **evidence** (≥2 figures) and a
  **caveat** (CXCL10 fragile / D3-sensitive); set maturity = candidate.
- **Exercises:** selection-to-finding (or promote chain), finding view (summary/
  evidence/caveats/maturity), context efficiency, effort-per-finding.
- **Ground truth:** finding cites the ISG set, notes the **5/6-donor** robustness
  and the **CXCL10/D3 caveat**; ≥2 evidence links; provenance intact.
- **Cost:** medium (≈6–10 runs).

### L2-TIME — timepoint trend
- **Goal:** "When does the response peak over the timecourse?"
- **Seed:** `monocyte_stim.csv` loaded.
- **Milestones:** ISG score per timepoint → trend figure + simple stats →
  **finding** "response peaks at 6h."
- **Exercises:** a second finding in the same thread (navigation between two
  findings), trend-analysis flow.
- **Ground truth:** finding states **peak at 6h** (monotone 0h<2h<6h); evidence
  figure attached.
- **Cost:** medium (≈5–8 runs).

### L3-REPL — cross-dataset replication
- **Goal:** "Does the ISG finding replicate in an independent cohort? If so, make
  a claim."
- **Seed:** project already containing the L2-IFN finding (or seeded); upload
  `monocyte_ref.csv`.
- **Milestones:** repeat the ISG-module analysis on `monocyte_ref` → figure →
  result → **lift to a claim** "ISG response replicates across cohorts," with
  **evidence drawn from both datasets**.
- **Exercises:** multi-dataset navigation, claim assembly across threads,
  cross-entity provenance, retrieval (find the original finding to attach).
- **Ground truth:** a `claim` exists supported by findings/results from **both**
  `monocyte_stim` and `monocyte_ref`; provenance spans both datasets.
- **Cost:** medium (≈8–12 runs).

### L3-OVERTURN — overturned finding
- **Goal:** "GENE_X looks induced by stimulation — confirm it before claiming."
- **Seed:** `monocyte_stim.csv`; a *premature* finding "GENE_X is stim-induced"
  already drafted (the driver may create it, or seed it).
- **Milestones:** robustness check (leave-one-donor-out / drop D6) → effect
  **vanishes** → recognize it's a **D6 (QC) artifact** → **supersede/retract**
  the GENE_X finding → record a **corrected claim** that excludes GENE_X and
  notes the artifact.
- **Exercises:** versioning/supersession of findings, contradictory-results
  handling, does the UI make retraction/correction discoverable and clean,
  consistency (no dangling/contradictory claims left).
- **Ground truth:** the GENE_X finding ends **superseded/retracted** (not
  active-and-contradicted); corrected claim present; D6 named as the cause.
- **Cost:** medium (≈8–12 runs).

### L4-REVIEW — reviewer response
- **Goal:** "Reviewer 2 wants a cell-cycle control for the ISG claim — add it and
  update the paper without breaking provenance."
- **Seed:** **pre-seeded** "submitted manuscript" project
  (`seed/submitted_manuscript.py`).
- **Milestones:** locate the affected claim/figure → run the requested control
  (cell-cycle regression) on `monocyte_stim` → new figure/result → **attach as
  added evidence / caveat** to the existing claim → update the manuscript
  narrative; old provenance preserved.
- **Exercises:** navigating an existing structured project, extending a claim,
  provenance integrity after edits, manuscript update, retrieval ("find the
  claim this reviewer point concerns").
- **Ground truth:** the target claim gains the new control as evidence (or a
  resolved caveat); no orphaned/dangling entities; manuscript references the
  update.
- **Cost:** medium-heavy (seed + ≈4–6 live runs).

### L4-SCALE — multi-year accretion & audit
- **Goal:** "(a) Find all the evidence behind claim C3. (b) Audit the manuscript
  for any statement that isn't actually supported."
- **Seed:** **pre-seeded** large project (`seed/large_project.py`): 4 datasets,
  ~30 figures, 6 findings, 3 claims, a manuscript — one claim with a missing
  evidence link.
- **Milestones:** use search/navigation to enumerate C3's evidence chain; then
  scan the manuscript/claims and flag the **unsupported** one.
- **Exercises:** **navigation-at-scale** (retrieval recall, time/actions-to-find),
  **clutter** (curated-vs-total, scan cost), **provenance integrity** auditing,
  Tier-1 audits on a packed tree, Tier-2 scroll/click cost. This is the headline
  navigation/clutter stress test — no live generation needed.
- **Ground truth:** agent retrieves the **complete, correct** evidence set for
  C3 (matches the seeded graph) and correctly identifies the **one unsupported
  claim**; measured against the known seeded structure.
- **Cost:** light at runtime (pre-seeded; mostly read/search/probe), heavier to
  author the seed once.

## Coverage & cost summary

| Scenario | Flow exercised | Headline metric | Mode | Cost |
|---|---|---|---|---|
| L1-QC | upload→plot→pin | actions-to-first-kept | live | light |
| L1-DONOR | promote→result | result effort + suggest quality | live | light |
| L2-IFN | finding w/ evidence+caveat | effort-per-finding, context | live | medium |
| L2-TIME | second finding | two-finding navigation | live | medium |
| L3-REPL | claim across 2 datasets | cross-dataset provenance/retrieval | live | medium |
| L3-OVERTURN | supersede + correct | versioning/consistency | live | medium |
| L4-REVIEW | extend existing paper | provenance-after-edit, retrieval | seed + live | med-heavy |
| L4-SCALE | navigate/audit at scale | recall, clutter, audit | pre-seeded | light runtime |

**Resource posture (v1):** L1–L2 are cheap and run every iteration; L3 runs
periodically; L4 is mostly read/probe over a fabricated graph (cheap at runtime).
All data synthetic, deterministic, < 150 KB total.
